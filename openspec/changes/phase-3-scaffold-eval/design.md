## Context

Phase 1 でベースライン、Phase 2 でスキャフォールドが完成。本 Phase では同一条件で両者を比較し、論文の主張「推論時の構造化で機械的失敗が解消される」が Bonsai-8B (1-bit) でも成立するかを検証する。

## Goals / Non-Goals

**Goals:**
- Phase 1 と同一タスクセットでスキャフォールド版を評価
- 達成率の Before/After 比較
- 失敗モード分布の変化分析(unmasking 効果)
- 可能であれば Ablation Study(Corrector only vs Full)

**Non-Goals:**
- プロンプト最適化の探索(Phase 2 で確定したプロンプトをそのまま使用)
- 新しいロールの追加
- 論文の Qwen3-8B 結果の再現

## Decisions

### 1. 同一評価ハーネスを再利用
**選択**: Phase 1 で構築したハーネスの LLM 呼び出し部分のみスキャフォールドに差し替え
**理由**: 比較の公平性。評価条件(タスクセット、最大ターン数、タイムアウト)を統一。

### 2. Ablation は時間に余裕があれば実施
**選択**: Corrector のみ(2 段)と Full(3 段)の比較
**理由**: 論文で最も重要な知見の一つ。ただし評価時間が 2 倍になるため、スコープとの兼ね合い。

## Risks / Trade-offs

- **[推論時間の増大]** → 3 役呼び出しで 2-3 倍。168 タスクなら 12-24 時間。対策: チェックポイント機構
- **[改善ゼロ]** → Bonsai の能力限界で改善が出ない可能性。この場合も「限界の分析」として記事化可能
