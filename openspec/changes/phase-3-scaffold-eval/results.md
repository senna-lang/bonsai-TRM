### スキャフォールドの効果分析

- **Summarizer 発動**: 1705 回 (全ターンの一部で発動)
- **Corrector 修正**: 1854 回 (約 28%のターンで何らかの修正)
- **コンテキスト超過解消**: 22 → 1 (21 件解消)

### 考察 (速報)

なぜ 1-bit モデルでここまで改善したか:
1. **ベースラインが極端に低かった** (0%) ため、改善幅が大きく取れる
2. **機械的失敗の比率が高かった** (コンテキスト超過 22 件) → Summarizer が直接解決
3. **Bonsai のコード生成能力自体は低くない** (Phase 1 でコード抽出率 73.1%) → Corrector が API スキーマを修正することで成功に至る
4. **temperature=0** のため Phase 1/3 で同じ出力が得られ、スキャフォールドの差分が純粋に見える
