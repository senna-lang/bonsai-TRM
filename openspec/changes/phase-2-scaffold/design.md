## Context

McClendon et al. (2026) の 3 役スキャフォールドは、同じ凍結モデルを 3 つの異なるシステムプロンプトで呼び出すことで、小型モデルの「機械的失敗」(認証漏れ、API スキーマ違反、無限ループ)を推論時に解消する手法。論文の Ablation Study では Corrector が最大の貢献を示した。

Bonsai-8B は Qwen3-8B の 1-bit 派生であり、コンテキスト長 6K 超で品質劣化する特性がある(bonsai-bankai 実測)。

## Goals / Non-Goals

**Goals:**
- 論文の 3 役(Summarizer / Agent / Corrector)を Bonsai 用に実装
- Corrector を先に完成させ、段階的に Summarizer を追加
- Bonsai のコンテキスト長制限に対応した閾値設定
- 1-2 タスクで end-to-end テスト

**Non-Goals:**
- 新規ロールの発明(論文の 3 役に忠実に従う)
- プロンプト最適化の網羅的探索(基本的に論文のプロンプトを Bonsai 用に微調整)
- 論文にない追加機能の実装

## Decisions

### 1. Corrector を先に実装する
**選択**: Step 1 で Corrector のみ(Agent → Corrector → AppWorld の 2 段構成)を完成
**理由**: 論文の Ablation Study で Corrector が最大貢献。最小工数で効果を確認できる。
**代替案**: 3 役同時実装 → 実装が複雑化するリスク

### 2. Summarizer の閾値とメッセージ保持数を Bonsai 用に調整
**選択**: 論文の閾値 (24,000文字 / 6,000トークン) を Bonsai 用に半減: **12,000文字 / 3,000トークン**。メッセージ保持数も論文の N=26, K=6 から **N=13, K=3** に半減。
**理由**: bonsai-bankai Phase 1 で 6K 超の品質劣化を確認済み。論文は Qwen3-8B の 12K/32K コンテキストを前提としており、Bonsai の有効コンテキスト長に合わせてスケールダウンする。N, K の最適値は Phase 0 のコンテキスト長観察に基づいて再調整する。

### 3. 構造化出力はプロンプトで強制する
**選択**: JSON / コードブロックのフォーマットをシステムプロンプトで強く指示
**理由**: Bonsai は構造化出力が不安定(bonsai-bankai 実測)。パースエラー時のフォールバックも実装。

### 4. 論文の Method セクションから自前実装する
**選択**: 論文 p.5-7 の記述（アーキテクチャ図 Figure 1、各 Tier の仕様、Eq. 2-3）に基づいて自前実装
**理由**: 論文リポジトリ (`Aimpoint-Digital/appworld`) にスキャフォールドコードは未公開（2026-04-15 時点）。公開されているのは vLLM 設定と軽微なパッチのみ。プロンプトの正確な文面は不明だが、論文に要件（入力・出力・保持すべき情報）が具体的に記述されており、実装に十分。
**リスク**: プロンプト文面の差異が結果に影響する可能性。ただし Bonsai ≠ Qwen3-8B なので、同一プロンプトでも結果は異なる。記事では「論文の Method セクションに基づく再実装」と明記する。

### 5. AppWorld の既存エージェントループを活用する
**選択**: `simplified_react_code_agent` の LLM 呼び出し部分をラップし、1 回の呼び出しを Summarizer → Agent → Corrector の 3 回に拡張する
**理由**: AppWorld 調査で判明したインターフェース: LLM は OpenAI chat completion 形式、エージェントループは messages → code extraction → world.execute() → observation の単純な ReAct パターン。スキャフォールドはこの LLM 呼び出しレイヤーのみに介入する。

## Risks / Trade-offs

- **[Bonsai の構造化出力が不安定]** → Corrector 自体が壊れた出力を生成する可能性。対策: パースエラー時はフォールバック(Agent 出力をそのまま使用)
- **[3 役呼び出しで推論時間が 2-3 倍]** → 168 タスクのスキャフォールド評価が 12-24 時間に延びる。対策: スコープ縮小の選択肢を保持
- **[論文のプロンプトが Bonsai に合わない]** → Qwen3-8B 用に最適化されたプロンプトが 1-bit モデルで機能しない可能性。対策: 最小限の微調整(指示の簡潔化、出力フォーマットの明示)
- **[プロンプト文面が不明]** → 論文にはプロンプトの要件は記載されているが、正確な文面は未公開。対策: 要件に基づいて自前で作成し、記事で「論文の Method セクションに基づく再実装」と明記する。Bonsai 用の調整も必要なため、完全同一プロンプトは元々適切ではない
